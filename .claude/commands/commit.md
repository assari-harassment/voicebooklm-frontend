---
description: 変更内容から日本語コミットメッセージを生成してコミット
---

以下のコマンドで現在の変更とブランチ名を確認してください：

```bash
git branch --show-current
git status
git diff --staged
git diff
```

確認した変更内容をもとに、以下のルールでコミットしてください：

## ルール

### タイトル行（1行目）

- 日本語で一文、簡潔に書く
- 何を変更したかが分かるように書く
- **最後にブランチ名から取得したissue番号を `#番号` の形式で付ける**
  - 例: `feature/42` → `#42`
  - 例: `fix/123` → `#123`

### 本文（3行目以降）

- 空行を1行挟んでから書く
- 箇条書き（`-`）で変更内容の詳細や変更背景を記載
- GitHubで「...」をクリックすると展開表示される

## コミットメッセージの形式

```
変更内容の概要 #issue番号

- 詳細1: 具体的な変更内容や理由
- 詳細2: 追加・修正したファイルや機能
- 詳細3: 背景や補足情報
```

## 例

```
ログイン機能を追加 #42

- JWTトークンによる認証を実装
- ログインフォームコンポーネントを作成
- 認証状態のグローバル管理をzustandで実装
```

## 手順

1. 必要なファイルを `git add` する
2. コミットメッセージを提案して確認を取る
3. HEREDOCを使って `git commit` を実行

```bash
git commit -m "$(cat <<'EOF'
タイトル #番号

- 詳細1
- 詳細2
EOF
)"
```
